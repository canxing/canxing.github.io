java8-lambda表达式

发表于 2018-01-04 | 分类于 java

lambda表达式

lambda表达式，实在java8中新增的特性，lambda表达式

匿名内部类

匿名内部类是用于创建一种不用声明类名称的类，这种类往往只是用一次，可以用”一个实现了某种接口的类”来描述，但是没有具体的名字(也就是引用名称)

//声明一个接口
interface InterfaceName{
    void method();
}
//使用匿名内部类创建一个实现了InterfaceName接口的类，但是他没有具体的类型
new InterfaceName() {
        @Override
        public void method() {

        }
};

匿名内部类是一种书写代码的简化方式，在有些时候想使用实现了某种接口，但是有只是用一次的情况时创建使用

ExecutorService executorService = Executors.newCachedThreadPool();
    executorService.execute(new Runnable() {
        @Override
        public void run() {
            System.out.println("这里可以开启一个线程");
        }
});

以上使用匿名内部类创建了一个任务(实现了Runnable接口的类)然后在线程中开启,关于匿名内部类的更多细节就不演示了，主要说明lambda表达式的用法

匿名内部类可以简化一部分开发，但是还不够简洁，为什么创建一个任务需要使用那么多的代码

executorService.execute(new Runnable() {
    @Override
    public void run() {
        System.out.println("这里可以开启一个线程");
    }
}

除了 System.out.println("这里可以开启一个线程");以外的其他代码都是不必要的，于是在java8中了更简单的方式实现，那就是lambda表达式

lambda表达式

lambda表达式在很多编程语言中都在使用lisp，python，c#等，其中在lisp之类的语言中使用最多。
如果使用lambda表达式来创建线程任务是什么样呢

ExecutorService executorService = Executors.newCachedThreadPool();
    executorService.execute(() -> {
        System.out.println("这里可以开启一个线程");
    });

lambda表达式的语法比较简单

(parameters) -> expression
(parameters) -> {statements;}

parameters表示传入的参数，-> 用于分隔参数和主体，expression和statements都是主体，expression只有一条语句时可以不用{}，多条语句时需要使用{},lambda表达式主体的最后一条语句默认为返回值，可以有返回值页可以没有返回值。

下面来演示以下lambda表达式的使用

    //Apple类，具有颜色了重量两个属性
    public class Apple {
        private int weight;
        private String color;
        public Apple(int weight, String color) {
            super();
            this.weight = weight;
            this.color = color;
        }
        @Override
        public String toString() {
            return "Apple [weight=" + weight + ", color=" + color + "]";
        }
    }

//初始化一个Apple类组成的链表
List<Apple> inventory = Arrays.asList(
        new Apple(10,"green"),
        new Apple(20,"red"),
        new Apple(12,"red"),
        new Apple(25,"green"));
/*
*要完成任务有
1.筛选出所有颜色为红色的苹果
2.返回所有苹果的重量，用列表表示
3.将所有苹果的重量加10
4.将苹果按重量从小到大排序
均使用lambda表达式完成
*/
//1
//1.1定义一个苹果筛选项，按照大多数书上的要求来说定义为一个苹果谓词
interface ApplePredicate{
    boolean test(Apple a);
}
//1.2写一个筛选成功时需要执行的操作，参数是要筛选项以及实现ApplePredicate对象
public static List<Apple> filterApple(List<Apple> apples,ApplePredicate predicate) {
    List<Apple> result = new ArrayList<>();
    for(Apple apple : apples) {
        if(predicate.test(apple)) {//这里调用了ApplePredicate的test方法，判断条件为真的苹果加入，这个类由外部传入
            result.add(apple);
        }
    }
    return result;
}
//1.3实现筛选条件
List<Apple> redApples = filterApple(inventory, (Apple apple) -> apple.getColor().equals("red"));
System.out.println(redApples);

这样就是筛选出了颜色为红色的所有苹果，如果ApplePredicate对象的条件再变只需要修改为
List<Apple> result = filterApple(inventory,(Apple apple) -> apple.getWeight() > 20);
就可以筛选除所有重量大于20的苹果

//2
//2.1定义一个接口方法，表示传入苹果对象可以返回苹果重量
interface AppleWeight{
    int weight(Apple a);
}
//2.2定义一个实现，传入一个苹果列表和实现AppleWeight的对象，然后遍历苹果列表，传入AppleWeight对象的weight对象，将返回的重量添加到返回列表中
public static List<Integer> appleWeight(List<Apple> apples,AppleWeight weight){
    List<Integer> result = new ArrayList<>();
    for(Apple apple : apples) {
        result.add(weight.weight(apple));
    }
    return result;
}
//2.3使用
List<Integer> appleWeights = appleWeight(inventory,(Apple apple ) ->apple.getWeight());
System.out.println(appleWeights);

//3
//3.1定义一个接口方法可以将苹果的重量增加
interface AppleWeightAdd{
    void addWeitht(Apple apple);
}
//3.2定义方法，将传入的苹果对象列表遍历，并依次通过AppleWeightAdd对象的addWeight方法
public static void addWeight(List<Apple> apples,AppleWeightAdd weight) {
    for(Apple apple : apples) {
        weight.addWeitht(apple);
    }
}
//3.3实现
addWeight(inventory,(Apple apple) -> apple.setWeight(apple.getWeight() + 10));
System.out.println(inventory);

//4.java8的list已经有了排序方法，只需要传入实现了Comparator的对象即可
inventory.sort((Apple a1,Apple a2) -> a1.getWeight() - a2.getWeight());
System.out.println(inventory);

关于lambda表达式的注意事项

lambda表达出现的位置必须是只有一个非默认接口才行，java8中添加了默认方法，可以在接口中定义方法体，查看Comparator接口可以看见有许多方法，带有default修饰符,这样的方法是默认方法
lambda表达式必须的参数和返回值必须和默认方法的参数和返回值相同
